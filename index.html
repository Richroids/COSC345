Editor

For our editor we are after a heavyweight, modern IDE that can support TypeScript. A heavyweight IDE provides error checking and find usage functionality that can act as early debugging detection alongside time-saving features such as code completion. These requirements narrow down the options to some of the more well known IDE's that have had plugins written for TypeScript, and then some more specific editors that are designed to support TypeScript.

Editor Pros Cons
Visual Studio Typescript is a Microsoft associated language and so it would appear logical that a Microsoft IDE would provide the most up to date support for TypseScript. Some internet users have tested to see what IDE gets the most immediate updates when a new version of TypeScript is released. (source) while visual studio does get them rapidly, other IDE's appear a bit faster. Also visual studio runs only on Windows which could make it diificult for group members with mac based hardware.
Atom Atom is a text editor which prides itself on user customisability while maintaining features and usability. It has implemented support for TypseScript updates (using an extension download) faster than all the other editors also and has cross platform support. None of our group members have heard of atom before this project and some believe that our time spent upskilling could be spent more valuably
Eclipse This is a modern, powerful IDE that all group memebers are somewhat familiar with. It has full TypseScript support with fast updates. It runs on windows and mac and additionally comes preinstalled on computers in Owheo. While one of two major IDE's that the University of Otago endorse, most group members are more familiar with netbeans. Eclipse also doesn't come with any native github repository support.
